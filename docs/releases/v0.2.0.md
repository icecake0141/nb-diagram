# v0.2.0

## Summary

`v0.2.0` is a major redesign release that moves the project to an API-first import workflow while keeping result-view/download compatibility.

## Breaking / Behavior Changes

- `POST /upload` is now deprecated and returns `410`.
- Primary flow is now:
  1. `POST /api/imports`
  2. `PUT /api/imports/{id}/mapping`
  3. `POST /api/imports/{id}/execute`
  4. `GET /api/graphs/{id}`
  5. `GET /api/exports/{id}`

## Added

- Import-run lifecycle API endpoints.
- Graph/export API endpoints.
- OpenAPI contract endpoint: `GET /api/openapi.yaml`.
- Database migration files and migration tracking (`schema_migrations`).
- Frontend module split (`app-main`, `diagram`, `import-workflow`).
- Frontend source tree for Vite + TypeScript (`frontend/src`).
- Sync/check scripts for environments without Node toolchain.
- CI frontend sync integrity check.

## Fixed

- drawio XML attribute escaping for quote characters.
- Role handling mismatch around `powered_device`.
- Type-check errors surfaced by CI (`mypy`).
- Style/format inconsistencies (`black`).

## Migration Notes

- If you relied on `/upload`, switch clients to the API workflow.
- Existing stored results remain accessible via `/result/<id>` and `/files/<id>/<kind>`.
- For frontend updates in environments without Node:
  - run `python3 scripts/sync_frontend.py`
  - verify with `python3 scripts/check_frontend_sync.py`
